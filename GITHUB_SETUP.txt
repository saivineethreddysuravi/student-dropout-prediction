# 📤 GITHUB SETUP GUIDE

Your project is now ready to push to GitHub! Follow these steps:

## Step 1: Create a GitHub Repository

1. Go to: https://github.com/new
2. **Repository name:** `student-dropout-prediction`
3. **Description:** Machine learning web app for predicting student dropout outcomes
4. **Public or Private:** Choose Public (great for portfolio)
5. **Initialize with README:** NO (we have our own)
6. Click **"Create repository"**

## Step 2: Copy the Repository URL

After creating, GitHub will show you a URL like:
```
https://github.com/yourusername/student-dropout-prediction.git
```

Or via SSH:
```
git@github.com:yourusername/student-dropout-prediction.git
```

## Step 3: Add Remote and Push

Open terminal and run:

```bash
cd /Users/vr/Desktop/Project

# Add the remote repository (replace YOUR-USERNAME)
git remote add origin https://github.com/YOUR-USERNAME/student-dropout-prediction.git

# Rename branch to main (recommended modern practice)
git branch -m master main

# Push to GitHub
git push -u origin main
```

## Step 4: Verify on GitHub

1. Go to your GitHub repository URL
2. You should see all your files there
3. The README should display automatically

## 🔒 Authentication

### If using HTTPS:
- GitHub will ask for username/password or personal access token
- Create token at: https://github.com/settings/tokens

### If using SSH (recommended):
1. Generate SSH key: `ssh-keygen -t ed25519`
2. Add to GitHub: https://github.com/settings/ssh/new
3. Use SSH URLs for push/pull

## ❗ Troubleshooting

### "fatal: remote origin already exists"
```bash
git remote remove origin
git remote add origin https://github.com/YOUR-USERNAME/student-dropout-prediction.git
```

### "Permission denied (publickey)"
- Use HTTPS instead of SSH
- Or set up SSH key correctly

### "Everything up-to-date"
- Your changes are already pushed
- Make new changes and commit again

### "Nothing to commit"
- No changes made since last commit
- Edit a file, then: `git add . && git commit -m "message"`
